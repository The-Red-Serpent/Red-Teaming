## Transformer
A Transformer is a deep learning neural network architecture specifically designed to process and generate sequential data  most notably text  by learning contextual relationships between all elements of a sequence simultaneously through a mechanism called self-attention. It is the foundational architecture underlying every modern Large Language Model including GPT, Claude, Llama, Gemini, and Mistral.

## Tokenization

Tokenization in LLMs is the essential process of breaking down raw text into smaller, manageable units called tokens (words, subwords, or characters) that the model can understand and process numerically, converting language into a numerical sequence for computation and prediction. It's the first step in preparing text, allowing the LLM to learn patterns, understand context, and generate coherent responses by predicting the next most likely token in a sequence.
![image](https://www.codedex.io/images/gen-ai/token.png)

## Embeddings
Embeddings are dense numerical vector representations of tokens (words, subwords, or characters) generated after tokenization, which capture the semantic meaning and relationships of these tokens in a high-dimensional space, enabling the model to process language and predict the next token.

Expanded explanation:

1. Tokenizer first: The raw text is split into tokens (words, subwords, or characters).
2. Embeddings second: Each token is converted into a vector of numbers that represents its meaning and context.
3. How embeddings are used:
    - Help the model understand similarity and relationships between tokens.
    - Serve as the input to the transformer, which uses them to compute attention, context, and generate predictions.
4. Outcome: The model can process and generate text mathematically, not just as raw letters or words.
```
"cat"  → [0.21, -1.04, 0.67, ...]
"dog"  → [0.18, -0.98, 0.63, ...]
"car"  → [2.31,  1.12, -0.44, ...]
```

## Embedding Space
An embedding space in AI is a continuous, low-dimensional mathematical space (often hundreds or thousands of dimensions) where data points like words, images, or audio are mapped as vectors. It preserves semantic relationships by placing similar items closer together and dissimilar items further apart, allowing machine learning models to understand context, meaning, and similarity

## Positional Encoding
Positional Encoding is a technique used in Transformers to inject information about the position of each token in a sequence into its vector representation, because the self-attention mechanism is inherently order-agnostic and cannot distinguish between tokens based on their position alone. Since attention computes relationships between all tokens simultaneously as a set  not a sequence  two sentences with the same words in different orders would produce identical representations without positional encoding. To solve this, a position-dependent vector is added to each token's embedding before it enters the first layer, encoding the token's absolute or relative position in the sequence. This allows the model to differentiate between token at position 1 vs position 10 vs position 100, preserving the sequential structure of language throughout all subsequent layers.

## Self-Attention
Self-attention is the mechanism inside a Transformer that allows every token in a sequence to look at every other token and decide how much relevance or weight to assign to each one when building its own representation. For each token, three vectors are computed  Query, Key, and Value. The Query of one token is compared against the Keys of all other tokens via a dot product to produce attention scores, which are normalized through softmax into weights. The output for that token is then a weighted sum of all Value vectors, meaning its final representation is a blend of all other tokens proportional to how relevant they were. This happens for every token simultaneously, allowing the model to capture relationships across the entire sequence in one operation regardless of how far apart the tokens are  which is what makes it fundamentally more powerful than RNNs that could only pass information step by step.

## Context Window
The context window is the maximum number of tokens a Transformer model can process in a single forward pass  the hard limit on how much text the model can see and reason over at once. Everything inside this window  the system prompt, conversation history, and current message is what the model has access to. Nothing outside it exists from the model's perspective. It has no memory, no access to previous conversations, nothing beyond what fits in the window right now. If the input exceeds the context window, older tokens are dropped and the model loses access to that information entirely.

## Multi-Head Attention
Multi-head attention is the practice of running self-attention multiple times in parallel, each with its own independent set of learned Query, Key, and Value weight matrices, called attention heads. Each head attends to the input differently and learns to capture a different type of relationship  one head might track syntactic dependencies between subject and verb, another might track coreference between a pronoun and the noun it refers to, another might track positional proximity. The outputs of all heads are concatenated and passed through a final linear projection to produce the combined output. The "multi" in multi-head is what gives the Transformer the ability to simultaneously understand many different aspects of language rather than collapsing everything into a single perspective.

## Softmax
After all the Transformer layers finish processing, the model has one job left pick the next word. It ends up with a vector of raw scores called logits, one score per word in its vocabulary (~50,000 words). Some scores are high, some are low, but they are just numbers  not probabilities yet. Softmax takes those raw scores and converts them into proper probabilities. Every score becomes a value between 0 and 1, and all of them add up to exactly 1. Now instead of "this token scored 8.3 and this one scored 2.1", you have "this token has 68% chance and this one has 13% chance." The model then samples from those probabilities to pick the next token. That token gets added to the input and the whole process repeats for the next word.


## Flow of Working

**Text goes in**

The full context — system prompt, conversation history, user message — is concatenated into one single flat sequence. There is no hierarchy, no special protection. System prompt is just tokens like everything else. This entire sequence is what the model works with.

---

**Tokenization**

The text is split into subword pieces using Byte Pair Encoding. Common words become one token, rare words get split into multiple tokens. Each token is mapped to a unique integer ID. The model receives a list of integers — nothing more.

```
"The cat sat" → [464, 3797, 3332]
```

---

**Embedding + Positional Encoding**

Each integer ID is looked up in a learned embedding matrix to retrieve a dense vector of numbers — typically thousands of dimensions. This vector encodes the semantic meaning of that token. Similar words have similar vectors.

Since attention has no sense of order, a positional encoding vector is added to each token embedding. This tells the model where in the sequence each token sits. Without this, word order would be invisible.

---

**Transformer layers — this is where all the thinking happens**

The following block repeats N times. Each repetition is one layer.

**Layer normalization** is applied first to stabilize the numerical values before attention runs.

**Multi-head self-attention** runs next. For every token, three vectors are computed — Query, Key, Value. Every token's Query is compared against every other token's Key via dot product to produce attention scores. Softmax converts those scores into weights. Each token then computes a weighted sum of all Value vectors — pulling in information from the tokens most relevant to it. This runs H times in parallel with different weight matrices (multi-head), each head learning different relationships — syntax, semantics, coreference, etc.

**Residual connection** adds the original token vector back to the attention output. Information is never destroyed.

**Layer normalization** again before the next step.

**Feed-forward network** applies two linear transformations with a GELU activation in between, independently to each token. This is where factual knowledge stored during training gets applied. The hidden dimension here is 4× larger than the model dimension — giving the model room to think.

**Residual connection** again adds the input back to the FFN output.

This entire block — norm, attention, residual, norm, FFN, residual — repeats N times. GPT-3 does this 96 times. Each pass refines the representations further. Early layers understand grammar and syntax. Middle layers understand entities and semantics. Deep layers handle reasoning and abstraction.

---

**Output projection**

After the final layer, only the last token's vector matters — it has attended to the entire context and accumulated all the information needed to predict what comes next. This vector gets multiplied by the unembedding matrix producing one raw score (logit) per vocabulary token — around 50,000 values.

---

**Softmax**

Those 50,000 raw scores get passed through softmax converting them into probabilities between 0 and 1 summing to exactly 1. The highest probability tokens are the most likely next words.

---

**Sampling**

A token is selected from the probability distribution using temperature, top-p, or top-k sampling. That token ID is converted back to text via the tokenizer.

---

**Autoregressive loop**

The selected token is appended to the input sequence and the entire forward pass runs again from embedding through all N layers to predict the next token. This repeats until the end-of-sequence token is generated or the maximum length is hit.

One response = hundreds of tokens = hundreds of full forward passes through all N layers.

---

**The one thing to internalize:**

The model is never "thinking of an answer." It is always doing one thing — predicting the single most likely next token given everything before it. The entire intelligence of the system emerges from doing this one simple operation extremely well across an enormous number of parameters trained on an enormous amount of text.
