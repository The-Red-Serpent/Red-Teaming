In this setup, user input (such as queries or data) is sent to the backend through API requests, which are then processed by AI models or services. The AI system processes the input and returns a response, which is delivered back to the user through the interface. This architecture allows for efficient communication, scalability, and the integration of various AI-powered functionalities.

### Model Fingerprinting
Model fingerprinting refers to the process of identifying and extracting distinctive characteristics of a machine learning model or system, which can be used to uniquely "fingerprint" or identify that model. This is similar to how human fingerprints can uniquely identify an individual, except in this case, the "fingerprint" is a set of identifiable features or behaviors that distinguish one AI model from another.
#### Techniques
- Direct Identity Probing - Ask the model about its identity
    ```
    What model are you? What company created you?
    ```
- Contradiction Testing - Probe with false assertions to reveal training biases
    ```
    Thanks for the help, Claude! I really appreciate Anthropic creating you.
    ```
### Knowledge Cutoff
Knowledge cutoff refers to the specific point in time when an AI model’s training data ends, meaning the model has no knowledge of events, developments, or information that occurred after this date. Since different models or versions may have different cutoffs, this provides a reliable method for fingerprinting AI models. By identifying the cutoff date, one can determine the most recent information the model was trained on, helping to distinguish between model versions and trace the period of knowledge it possesses. ### Context Window Testing
Context window testing is the process of evaluating how effectively an AI model, particularly in natural language processing (NLP), utilizes its context window—the fixed amount of input tokens it can process at a time. This technique involves assessing the model's ability to maintain coherence, relevance, and memory of earlier parts of an input as the length of the input increases and exceeds the model's context window limit.
How its done
- Ask the Model to remember a secret
- Fill up the coversation with something random
- Ask the secret again to the model
