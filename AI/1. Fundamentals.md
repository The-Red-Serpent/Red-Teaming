### AI
Artificial Intelligence (AI) is the class of computational systems designed to perceive, learn from data, reason under uncertainty, and take actions to achieve specified objectives without being explicitly programmed for every scenario.

### Transformers
Transformers are a type of neural network architecture used in AI that processes and understands data by focusing on relationships between all parts of the input at once, rather than step by step. They are especially effective for language tasks and form the foundation of modern large language models.



### Generalization
Generalization refers to the model's ability to accurately predict outcomes for new, unseen data not used during training. A model that generalizes well can effectively apply its learned knowledge to real-world scenarios.

### Overfitting
Overfitting is a situation in machine learning where a model learns the training data too closely, including noise and outliers, resulting in high accuracy on training data but poor performance on new, unseen data.
### Underfitting
Underfitting is a situation where a model is too simple to capture the underlying patterns in the data, leading to poor performance on both training and unseen data

### Cross-Validation
Cross-validation is a technique in machine learning used to assess how well a model will generalize to new, unseen data. It involves splitting the dataset into multiple subsets (folds), training the model on some folds, and testing it on the remaining fold. This process is repeated multiple times, and the results are averaged to provide a reliable estimate of the model’s performance.

### Regularization
Regularization is a technique that helps AI models not get too obsessed with the training data. It makes the model focus on the big patterns instead of memorizing every tiny detail, so it can perform well on new, unseen data.
#### Types
- L1 (Lasso): Makes the model ignore unnecessary features—keeps only the important stuff.
- L2 (Ridge): Makes the model play it safe, keeping all features but not letting any one feature dominate.

### Orchestration Framework
An orchestration framework in AI is a software system that coordinates and manages the flow of tasks between multiple components (like models, tools, data sources, and APIs), ensuring they execute in the correct order, share data properly, and adapt based on logic or outcomes.

Examples of orchestration frameworks

LangChain
  - Used to build step-by-step pipelines connecting LLMs with tools, memory, and data.
  
LangGraph
  - Designed for more complex, stateful workflows with loops, branching, and decision-making.
  
LlamaIndex
  - Helps orchestrate how data is retrieved and passed into language models.
  
Apache Airflow
  - General-purpose workflow orchestrator often used in ML pipelines and data engineering.
  
Prefect
  - Another tool for managing and scheduling data and ML workflows.




### Inference Server
An Inference Server is a service or software that handles the execution of machine learning models, providing an environment where the model can receive input data, process it, and produce predictions or outputs. It's specifically designed to manage the workload of running models in a production or operational environment.

### Inference Server Host

The Inference Server Host refers to the machine (physical or virtual) where the inference server is running. It’s the hardware infrastructure or environment that houses the inference server, providing resources like CPU, GPU, memory, and storage to support model inference.

### Model Inference
Model Inference (or just inference in AI) refers to the process where a trained AI model makes predictions or decisions based on new, unseen data. In simpler terms, it's when the model applies the knowledge it gained during training to solve real-world problems or provide outputs for user input.

### Parameters

Parameters are the weights and biases that the model learns during the training process. These parameters are adjusted through training to minimize the error between the model's predictions and the actual outcomes (for example, in supervised learning tasks like image classification or text generation). The goal of training is to find the optimal values for these parameters so that the model can make accurate predictions on new, unseen data.

Breakdown of Parameters:
Weights: These are the parameters that define the strength of the connection between neurons in the network. During training, the model adjusts these weights to learn patterns in the data.
Biases: These are additional parameters that allow the model to adjust its output independently of the input data, giving it more flexibility.

The more parameters a model has, the more complex patterns it can potentially learn. A larger model with billions of parameters can capture subtle relationships in data that smaller models with fewer parameters might miss.


### System Prompt 

A system prompt in the context of AI refers to the initial input or instruction given to an AI model that guides its behavior and responses. It acts as a setup or configuration that defines the model's role, style, or the kind of task it should focus on. Essentially, a system prompt helps shape how the AI interacts with users or performs its tasks by setting the context and parameters for its responses.

Guardrails in AI are a set of predefined rules, constraints, and validation mechanisms applied to AI systems to control their inputs, outputs, and actions, ensuring the system behaves safely, accurately, and within intended ethical, legal, and operational boundaries. ### Knowledge Cutoff
Knowledge cutoff in AI is the specific date up to which a language model has been trained on data, representing the latest point in time for which it has built-in knowledge of facts, events, and information.
