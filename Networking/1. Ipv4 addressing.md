### Ipv4 Address Classes:
IPv4 addresses are divided into **five classes (A, B, C, D, and E)** based on their first octet:

|Class|First Octet Range|Default Subnet Mask|Network Bits|Host Bits|Max Networks|Max Hosts per Network|
|---|---|---|---|---|---|---|
|A|1 - 126|255.0.0.0|8|24|128|16,777,214|
|B|128 - 191|255.255.0.0|16|16|16,384|65,534|
|C|192 - 223|255.255.255.0|24|8|2,097,152|254|
|D|224 - 239|N/A (Multicast)|N/A|N/A|N/A|N/A|
|E|240 - 255|N/A (Experimental)|N/A|N/A|N/A|N/A|

- **Class A**: Large networks (e.g., ISPs, large organizations).
- **Class B**: Medium-sized networks.
- **Class C**: Small networks (e.g., home, small businesses).
- **Class D**: Used for **multicasting**.
- **Class E**: Reserved for experimental use.

### Types of IPv4 Addresses:
#### Unicast Address:
**Definition**: A unicast address is used for one-to-one communication between a source and a single destination.
#### Broadcast Address
**Definition**: A broadcast address is used for one-to-all communication, sending data to **all hosts** in a network.

#### Multicast Address
**Definition**: A multicast address is used for one-to-many communication, sending data to **multiple devices** in a specific group.


### Subnetting:
Subnetting divides a large network into smaller sub-networks for better efficiency.

Every IP address consists of:

- **Network Portion:** Identifies the network.

#### Splitting the Network

Step 1: Determine the New Prefix

1. A **/24** subnet has **256 IPs** (`2^8`).
    
2. To divide it into **4 equal subnets**, we need to find the number of **extra bits** required for subnetting.
    
3. The formula for subnet division is:
    
    2n=Number¬†of¬†Subnets2^n = \text{Number of Subnets}
    - We need **4 subnets**, so:
        
        2^n=4
        2^n = 4 
        n=2
    - Since the original subnet mask was **/24**, adding 2 extra bits gives us:
        
        /24+2=/26/24 + 2 = /26
    - **New subnet prefix = /26**
        
    - This means the subnet mask changes from `255.255.255.0` (for /24) to `255.255.255.192` (for /26).

#### Fixed Length Subnet Mask
It is a networking methodology that divides the block of Ip addresses into multiple subnets of equal lengths or equal no of addresses. All subnets have same subnet mask and same no of hosts

### (A) Understanding FLSM Subnetting

- A network is divided into **equal-sized subnets**.
- The subnet mask is **fixed** for all subnets.
- IPs are often **wasted** if a subnet doesn‚Äôt need all assigned IPs.

1. **192.168.1.0/24 has 256 IPs (usable: 254).**

3. **Formula:**
    
    Subnet¬†Calculation=2^h‚â•Number¬†of¬†Subnets
    
    
    Where **h** is the number of borrowed bits from the host portion.
    
4. **2¬≤ = 4** ‚Üí Borrow **2 bits** ‚Üí **New subnet mask = /26 (255.255.255.192)**

#### Variable Length Subnet mask
**VLSM (Variable-Length Subnet Masking)** allows the use of **different subnet masks within the same network**, making IP allocation **more efficient** compared to Fixed-Length Subnet Masking (**FLSM**).

####  How Does VLSM Work?

##### **Step 1: Identify Subnet Requirements**

Each subnet requires a **different number of hosts**, so we assign the smallest subnet mask that fits the requirement.

##### **Example Scenario:**

A company has been assigned **192.168.1.0/24** and needs subnets for:

1. **50 hosts**
2. **25 hosts**
3. **10 hosts**
4. **5 hosts**

---

##### **Step 2: Find the Best Subnet Mask for Each Subnet

Use the **Usable Hosts Formula** to determine the required subnet size:

Usable¬†Hosts=2h‚àí2\text{Usable Hosts} = 2^h - 2

Where **h** = number of host bits.
