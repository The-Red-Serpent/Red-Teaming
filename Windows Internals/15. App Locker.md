## Applocker
AppLocker is an application control technology that is built into Windows.  Its aim is to prevent users from running applications, scripts, or packages that are not approved by policy.  A policy contains one or more enforcement rules, where each rule contains a permission and a condition.  A permission defines the action (allow or deny), and the user or group to which the rule will apply.  A condition defines the rule itself that the policy will apply.  

This can be based on:

* Publisher - create a rule based on the publisher of a signed application.
* Path - create a rule based on a file or folder path.
* File hash - create a rule based on the hash of a file.

AppLocker policies are primarily stored in two places depending on how they are configured:

1. In the Local or Domain Group Policy

AppLocker rules are stored as XML policy data inside the Group Policy object.

For local policies, the configuration path is:
```
Computer Configuration
└── Windows Settings
    └── Security Settings
        └── Application Control Policies
            └── AppLocker
```

For domain-managed policies, the rules are stored inside the corresponding Group Policy Object (GPO) on the domain controller.

2. In the Registry

The active AppLocker policy is stored under the Windows Registry at:
```
HKLM\SOFTWARE\Policies\Microsoft\Windows\SrpV2
```

Under SrpV2 you'll typically find subkeys for each rule collection:

```
HKLM\SOFTWARE\Policies\Microsoft\Windows\SrpV2\Exe
HKLM\SOFTWARE\Policies\Microsoft\Windows\SrpV2\Dll
HKLM\SOFTWARE\Policies\Microsoft\Windows\SrpV2\Msi
HKLM\SOFTWARE\Policies\Microsoft\Windows\SrpV2\Script
HKLM\SOFTWARE\Policies\Microsoft\Windows\SrpV2\Appx
```

Each rule is stored as a GUID-named subkey containing the rule metadata and conditions.

Example:
```
HKLM\SOFTWARE\Policies\Microsoft\Windows\SrpV2\Exe\{Rule-GUID}
```

## Enumeration

### PowerShell:
```
# Check if AppLocker is enabled
Get-AppLockerPolicy -Local | Select-Object -ExpandProperty RuleCollections

# View all rules in detail
Get-AppLockerPolicy -Local | ForEach-Object {
    $_.RuleCollections | ForEach-Object {
        Write-Host "`n=== $($_.RuleCollectionType) ===" -ForegroundColor Cyan
        $_.Rules | Format-Table Name, Action, User, Conditions -AutoSize
    }
}

# Check effective policy (including GPO)
Get-AppLockerPolicy -Effective | Select-Object -ExpandProperty RuleCollections
```
