## Windows Registry
The registry is a central hierarchical database used to store information necessary to configure the system for one or more users, applications, and hardware devices. The data is structed in a tree format. Each node in the tree is called a key and each key can contain both subkeys and data entries called values which when grouped together is often referred to as a Hive. With this in mind, a Registry Hive is a group of keys, subkeys, and values stored in a single file on the disk.

Most system hives are stored at:
```
C:\Windows\System32\config
```

## Windows Registry Root Keys


| Root Key                         | Purpose                                                        | Scope                | Backing Hive/File                                                | Example Contents                                                    |
| -------------------------------- | -------------------------------------------------------------- | -------------------- | ---------------------------------------------------------------- | ------------------------------------------------------------------- |
| **HKLM** (`HKEY_LOCAL_MACHINE`)  | Stores settings for the computer itself                        | Entire system        | `SYSTEM`, `SOFTWARE`, `SAM`, `SECURITY`, `DEFAULT`               | Drivers, services, installed software, local users, security policy |
| **HKCU** (`HKEY_CURRENT_USER`)   | Stores settings for the currently logged-in user               | Current user only    | `NTUSER.DAT`                                                     | Desktop wallpaper, taskbar settings, application preferences        |
| **HKU** (`HKEY_USERS`)           | Contains registry data for all loaded users                    | All loaded users     | Multiple `NTUSER.DAT` files                                      | User profiles represented by SIDs                                   |
| **HKCR** (`HKEY_CLASSES_ROOT`)   | Stores file associations and COM registrations                 | System + User        | Derived from `HKLM\Software\Classes` and `HKCU\Software\Classes` | `.txt`, `.pdf`, `CLSID`, COM objects                                |
| **HKCC** (`HKEY_CURRENT_CONFIG`) | Stores information about the currently active hardware profile | Current boot session | Derived from `HKLM\SYSTEM`                                       | Display settings, printer configuration, hardware profile           |

## Registry ACL
An Access Control List (ACL) in the Windows Registry is a set of permissions attached to a registry key that defines which users, groups, or processes are allowed or denied access to perform operations such as reading, modifying, creating, or deleting registry data.
