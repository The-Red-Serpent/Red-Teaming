### Thread
A thread is the smallest unit of execution within a Windows process. It represents a single flow of control or sequence of instructions that the CPU executes independently, while sharing the process’s resources such as virtual memory, handles, and loaded modules.

                              |  +----------------------------------------------------+  |
                              |  | Thread Environment Block (TEB)                     |  |

### Components of a Thread:
Every thread has its own private set of things that belong only to it:

#### Registers 
The CPU has a set of small storage locations called registers — they hold the values the CPU is currently working with. Each thread has its own snapshot of these. When the CPU switches from one thread to another it saves the current thread's registers and loads the next thread's registers.

#### Stack
A stack is a memory region used to store local variables, function call information, and return addresses for the thread’s execution. Every thread has 2 stacks. when your thread makes a syscall and crosses into Ring 0, the kernel cannot trust or use the user mode stack  it needs its own separate, secure stack. So the kernel maintains a separate kernel stack for every thread.
  - User mode stack
  - Kernel mode stack

#### Priority
Every thread has a priority from 0 to 31. This tells the scheduler how important this thread is compared to others. Higher priority threads get to run first.

#### State
Indicates the current status of the thread, which can be one of the following: running, ready, waiting, or terminated.

```

| Thread State           | Meaning                                                                                        |
| -----------------------| ---------------------------------------------------------------------------------------------- |
| Init                   | Thread is newly created and is initializing.                                                   |
| Ready / Deferred Ready | Thread is ready to run but is waiting for CPU availability.                                    |
| Standby                | Thread has been selected by the scheduler and is next to run (context switch about to happen). |
| Running                | Thread is currently executing on a CPU core.                                                   |
| Waiting                | Thread is blocked waiting for an event, kernel object, I/O, etc. No CPU usage.                 |
| Transition             | Thread is ready to run but its kernel stack or resources are being paged back into memory.     |
| Terminate              | Thread has finished execution and is being cleaned up.                                         |

```

#### Thread-Local Storage (TLS)

A mechanism that allows each thread in a process to have its own private copy of a variable. Normally a global variable is shared across all threads  every thread reads and writes the same memory location, which causes race conditions. TLS solves this by giving each thread its own separate instance of that variable stored in a dedicated slot inside the thread's TEB, so threads never interfere with each other's data even when using the same variable name.

#### Scheduling Information 

Data that contains the thread’s priority, execution time (quantum), and state, used by the scheduler to determine when and how the thread is executed.

#### Thread Environment Block
A data structure that exists in user mode memory for every thread. It stores per-thread information that needs to be accessible from Ring 3 without making a syscall. The kernel creates and populates it when a thread is created. On x64 it is always accessible via the GS segment register meaning any code can read its own TEB instantly without asking the kernel for anything.



### Thread Scheduling
Thread scheduling in Windows is the mechanism used by the kernel to decide which thread gets to execute on a CPU core at any given time. Since multiple threads can be ready to run but there are limited CPU cores, the operating system must manage execution fairly and efficiently. This is handled by the Windows scheduler, which continuously selects a ready thread, assigns it CPU time, and switches between threads as needed.
